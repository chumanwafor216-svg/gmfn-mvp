# GMFN Innovation Trail (Pilot Phase)

## Core Breakthrough
**TrustSlip** — a verified, portable trust limit representing community-backed integrity expressed as a currency amount.

TrustSlip is not a bank guarantee and does not auto-debit guarantors in MVP. Instead, it:
- authorizes goods release based on visible social backing and audit history
- records actions as TrustEvents for explainable trust scoring
- supports enforceable network consequences upon default (reputation and access impacts)

## Deterministic Trust Engine
Trust in GMFN is deterministic:
- derived from an append-only TrustEvent ledger
- no hidden score adjustments
- repayment-only trust growth policy in MVP (e.g., +0.10 borrower, +0.03 guarantor as policy-aligned explainability)

## Explainability Surface
Implemented:
- `/trust/me/why` for user-friendly explainability
- `/admin/trust/why/{user_id}` for audit/evidence review
- Admin recent TrustEvents feed for screenshot-ready evidence (includes meta.reason/meta.note where present)

## Evidence Pack (Pilot-Ready)
Evidence pack meta and zip include:
- deterministic pack_id (pilot-safe)
- based_on_event_at (latest TrustEvent anchor)
- checksum (integrity anchor)
- embedded trust explainability payload and timeline snapshot
- TrustSlip evidence PDF and timeline PDF (where enabled)

## Merchant Verification (Low-End Friendly)
Merchant verification designed for low-end phones:
- public verify JSON (minimal disclosure)
- public HTML verify page (no JS)
- lite verification page (text-first)
- optional QR to verify link
- rate limited public endpoints to reduce guessing/abuse

## Lifecycle Enforcement
Admin-only lifecycle controls with TrustEvent audit:
- revoke / freeze / unfreeze
- extend expiry
- log merchant release events

## Pilot Safety Positioning
- GMFN is non-custodial in MVP
- does not process funds
- does not auto-debit guarantors
- focuses on transparency, auditability, and trust transmission

## Next Phase Candidates
- signed merchant verification tokens (HMAC) with strict secret requirements
- public/minimal verify endpoint using signed token (rate limited)
- expand evidence pack PDFs to include trust-why excerpts
- migrate rate limiting to Redis for multi-worker deployment
