The Trust Infrastructure Model
The purpose of GSN is not simply to describe trust.The purpose is to make trust visible, portable, and usable.The Trust Infrastructure Model explains how this happens.The model operates through five connected layers.LAYER 1 — COMMUNITIESCommunities create value through participation, contribution, accountability, service, leadership, cooperation, and support.Markets create trust.Churches create trust.Cooperatives create trust.Diaspora communities create trust.Apprenticeship systems create trust.This is where trust originates.LAYER 2 — TRUST EVIDENCETrust becomes useful when it becomes visible.Trust Events capture evidence.Community Records preserve evidence.Verification supports evidence.This transforms trust from memory into structured information.LAYER 3 — TRUST IDENTITYTrust Passport preserves demonstrated value.TrustSlip communicates demonstrated value.Trust therefore becomes portable.People no longer lose all recognition simply because they move to a new place.LAYER 4 — DECISION SUPPORTTrust evidence must be usable.Market Wisdom improves judgement.Community Verification improves confidence.The Command Centre improves visibility and understanding.Trust evidence becomes decision support.LAYER 5 — OPPORTUNITYThe final purpose of trust infrastructure is opportunity.Trust supports:• Employment• Trade• Finance• Support• Participation• Community GrowthTrust becomes opportunity.The Trust Infrastructure Model can therefore be summarized as:Communities↓Trust Events↓Trust Evidence↓Trust Passport↓TrustSlip↓Verification↓Decision↓OpportunityThis is the operational heart of GSN.It explains how community-created value becomes visible, how visibility becomes recognition, and how recognition becomes opportunity.
