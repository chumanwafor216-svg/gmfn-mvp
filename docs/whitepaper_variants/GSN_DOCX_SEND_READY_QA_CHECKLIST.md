# GSN DOCX Send-Ready QA Checklist

Last updated: 2026-07-09

Use this before sending any white paper copy externally. This checklist exists because the generated DOCX files were structurally checked, but not visually rendered in this environment.

## Files To QA

```text
[ ] GSN_Trust_Infrastructure_RGU_Academic_Copy_Chris_Mulley.docx
[ ] GSN_Customer_Discovery_Plain_Language_Copy.docx
[ ] GSN_Trust_Infrastructure_Investor_Brief.docx
```

Do not send the original source white paper unless it has been separately cleaned.

## First Open Check

Open each file in Word or Google Docs.

```text
[ ] File opens without repair warning.
[ ] Title appears correctly.
[ ] Body text appears normal, not compressed or broken.
[ ] Bullets and numbered lists appear as real lists.
[ ] Page breaks do not create awkward blank pages.
[ ] No text runs into margins.
[ ] No heading sits alone at the bottom of a page.
```

If Word asks to repair the file, do not send it. Export a clean copy from Word or Google Docs first.

## Audience Fit Check

### Chris / RGU Academic Copy

```text
[ ] Sounds like academic/customer discovery scrutiny, not a sales pitch.
[ ] Mentions RGU as a review/discovery context, not an endorser.
[ ] Explains safeguards: consent, correction, privacy, provenance, human judgement.
[ ] Makes the ask clear: guidance on structured customer discovery.
[ ] Does not lead with visa pressure.
```

### Customer Discovery Plain-Language Copy

```text
[ ] A normal participant can understand it without theory background.
[ ] It asks for honest feedback, not agreement.
[ ] It avoids heavy Behavioural Capital language.
[ ] It explains the social-scoring risk in plain language.
[ ] It makes privacy and refusal feel allowed.
```

### Investor Brief

```text
[ ] Clearly says the thesis still needs discovery/pilot proof.
[ ] Does not claim product-market fit.
[ ] Does not imply GSN is a bank, lender, credit scorer, or guarantee engine.
[ ] Names risks and proof still needed.
[ ] Keeps the investor ask focused on proof requirements, not immediate certainty.
```

## Forbidden Claim Check

Search or manually scan for these risky claims.

```text
[ ] No claim that RGU endorsed GSN.
[ ] No claim that the theory is proven.
[ ] No claim that product-market fit exists.
[ ] No claim that the first wedge is proven.
[ ] No claim that privacy/governance is solved.
[ ] No claim that GSN approves credit or guarantees trust.
[ ] No claim that Behavioural Capital is an accepted market category.
```

## Export Check

If sending PDF instead of DOCX:

```text
[ ] Export from Word or Google Docs after visual check.
[ ] Open the exported PDF.
[ ] Confirm page count.
[ ] Confirm links, bullets, and headings survived export.
[ ] Confirm the filename still matches the audience.
```

If sending DOCX:

```text
[ ] Save once from Word or Google Docs after visual check.
[ ] Reopen the saved file.
[ ] Confirm there is no repair warning.
```

## Attachment Check

Before pressing send:

```text
[ ] Correct recipient.
[ ] Correct attachment.
[ ] Correct email draft.
[ ] No investor brief attached to Chris/RGU email.
[ ] No academic copy attached to normal participants.
[ ] No internal guide or meeting brief accidentally attached.
[ ] Filename is professional and readable.
```

## Final Human Read

Read the first page and the final page of each document.

```text
[ ] First page makes the audience clear.
[ ] First page does not overclaim.
[ ] Final page ends with an ask or next step.
[ ] The document feels shorter than the reader's patience.
```

## Devil's Advocate

A badly formatted or overclaiming document can damage a good idea faster than no document at all. If the document looks rough, send a shorter email and ask for a conversation instead. The goal is not to prove sophistication through volume; it is to earn the next serious conversation.
